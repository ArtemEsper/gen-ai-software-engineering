## ЁЯУЛ Features

### Core Endpoints
| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/transactions` | Create a new transaction |
| `GET` | `/transactions` | List all transactions (with filtering) |
| `GET` | `/transactions/{id}` | Get a specific transaction by ID |
| `GET` | `/accounts/{accountId}/balance` | Get account balance |
| `GET` | `/accounts/{accountId}/summary` | Get account transaction summary тнР |

### Validation
- **Amount**: Must be a positive number with at most 2 decimal places
- **Account format**: Must match `ACC-XXXXX` (5 alphanumeric characters after the dash)
- **Currency**: Must be a valid ISO 4217 code (USD, EUR, GBP, JPY, CAD, AUD, CHF, CNY, SEK, NZD, MXN, SGD, HKD, NOK, KRW, TRY, INR, BRL, ZAR, DKK)
- **Type**: Must be one of `deposit`, `withdrawal`, or `transfer`

### Transaction Filtering (`GET /transactions`)
| Query Parameter | Example | Description |
|----------------|---------|-------------|
| `accountId` | `?accountId=ACC-12345` | Filter by account (matches sender or receiver) |
| `type` | `?type=transfer` | Filter by transaction type |
| `from` | `?from=2024-01-01` | Filter transactions on or after this date |
| `to` | `?to=2024-01-31` | Filter transactions on or before this date |

Filters can be combined freely.

### Extra Feature: Account Summary (Option A)
`GET /accounts/{accountId}/summary` returns:
- `total_deposits` тАФ sum of all deposit amounts into the account
- `total_withdrawals` тАФ sum of all withdrawal amounts from the account
- `transaction_count` тАФ total number of transactions involving the account
- `most_recent_transaction` тАФ ISO 8601 timestamp of the latest transaction

## ЁЯПЧя╕П Architecture & Design Decisions

### Technology Stack
- **FastAPI** тАФ chosen for automatic OpenAPI/Swagger docs, built-in Pydantic validation, and minimal boilerplate
- **Pydantic v2** тАФ handles request validation and response serialization
- **Uvicorn** тАФ ASGI server to run the application
- **Python standard library** тАФ `uuid`, `datetime`, `re` for ID generation, timestamps, and regex validation

### Storage
All transactions are stored in a module-level Python dictionary (`dict[str, Transaction]`) keyed by transaction ID. Data is lost when the server restarts тАФ this is intentional per the assignment requirements.

### Balance Calculation
The balance for an account is computed on-the-fly from all stored transactions:
- **Deposits** where `toAccount` matches тЖТ add to balance
- **Withdrawals** where `fromAccount` matches тЖТ subtract from balance
- **Transfers** where `toAccount` matches тЖТ add to balance
- **Transfers** where `fromAccount` matches тЖТ subtract from balance

### HTTP Status Codes
| Code | When |
|------|------|
| `200` | Successful GET |
| `201` | Transaction created successfully |
| `400` | Validation error (invalid input) |
| `404` | Transaction or account not found |

### Auto-generated Fields
When a transaction is created via `POST /transactions`, the API automatically sets:
- `id` тАФ a UUID v4 string
- `timestamp` тАФ current UTC time in ISO 8601 format
- `status` тАФ set to `"completed"`
